# 09 — Generated vs Authored Docs

> Người viết "tại sao", máy sinh "cái gì". Khi data phình to (migrations, routes, content), doc liệt kê viết tay sẽ stale — phải sinh tự động từ source of truth.
> *(EN: Humans write the "why", machines generate the "what". As data grows, hand-written inventory docs go stale — generate them from the source of truth.)*

---

## Vấn đề / The problem

App-map viết tay hoạt động tốt khi project nhỏ. Khi phình to (trigger: DB > 15 migrations / app > 50 routes — xem bảng scale-up ở methodology README):
- DB lên hàng chục migrations → bảng schema trong doc viết tay lệch sau mỗi migration
- App lên 100+ routes → danh sách page trong `01-pages.md` thiếu dần
- Content lên hàng nghìn item (bài học, đề thi, bài viết) → con số thống kê trong doc sai ngay tuần sau

Người không nên — và không thể — maintain loại doc này bằng tay. Nhưng AI lại **rất cần** chúng để khỏi grep toàn codebase.

---

## Giải pháp / The solution

Chia app-map thành 2 loại có lifecycle khác nhau:

```
docs/app-map/
├── 01-flows.md                 # AUTHORED — người viết
├── 03-database-design.md       # AUTHORED — tại sao schema như vậy, invariant, RLS strategy
├── ...
└── _generated/                 # GENERATED — máy sinh, người KHÔNG sửa
    ├── schema.md               # Dump từ DB thật (tables, columns, indexes, RLS policies)
    ├── routes.md               # Quét từ src/app/ hoặc router config
    ├── content-stats.md        # Đếm từ data thật (số bài, số đề, coverage %)
    └── env-vars.md             # Quét từ .env.example + chỗ đọc env trong code
```

**Phân loại nội dung:**

| Loại | Ai viết | Ví dụ | Stale khi nào |
|---|---|---|---|
| WHY — quyết định, invariant, trade-off | Người (authored) | "RLS owner-only vì X", ADR, flow nghiệp vụ | Khi đổi quyết định (hiếm) |
| WHAT — inventory, schema, danh sách | Máy (generated) | Bảng columns, list routes, content counts | Mỗi commit (nên máy phải sinh) |

---

## Quy tắc cứng / Hard rules

1. **File generated có header bất khả xâm phạm**: dòng đầu là `<!-- AUTO-GENERATED by scripts/gen-docs — DO NOT EDIT. Regenerate: npm run gen-docs -->`
2. **Authored doc KHÔNG paste bảng inventory** — link sang `_generated/` thay vì copy (copy = stale sau 1 tuần)
3. **Sinh lại trong pre-commit hoặc CI** — không dựa vào "nhớ chạy script"; gắn vào hook của nguyên tắc 08
4. **Generated từ source of truth thật** — schema dump từ DB/migrations, routes quét từ filesystem; KHÔNG sinh từ doc khác
5. **Diff được review** — generated doc vẫn commit vào git để AI đọc được và diff hiện trong PR (thấy schema đổi gì ngay trong review)

## Ví dụ script tối thiểu / Minimal generator examples

```bash
# Schema từ Supabase (Postgres)
supabase db dump --schema public --data=false > /tmp/schema.sql
# → parse thành markdown table, hoặc đơn giản: copy SQL vào _generated/schema.md

# Routes từ Next.js App Router
find src/app -name "page.tsx" | sed 's|src/app||; s|/page.tsx||' > _generated/routes.md

# Content stats
psql -c "SELECT status, count(*) FROM lessons GROUP BY status" >> _generated/content-stats.md
```

Script không cần đẹp — cần **chạy tự động và đúng**.

## Anti-patterns

| Anti-pattern | Hậu quả |
|---|---|
| Sửa tay file trong `_generated/` | Lần sinh tiếp theo mất sạch sửa đổi |
| Paste schema table vào authored doc "cho tiện" | Stale sau 1 migration, AI đọc bảng cũ |
| Generator đọc từ doc khác thay vì source thật | Sinh ra rác đã stale sẵn |
| Không commit generated docs ("vì sinh được mà") | AI session mới không đọc được; PR không thấy diff schema |
| Sinh quá nhiều (cả file > 3K tokens không ai load) | Token waste — chỉ sinh cái AI thật sự cần tra |

## Checklist áp dụng / Adoption checklist

- [ ] Folder `docs/app-map/_generated/` tồn tại, có ít nhất schema + routes
- [ ] Mỗi file generated có header AUTO-GENERATED + lệnh regenerate
- [ ] Generator chạy trong pre-commit hook hoặc CI (không chạy tay)
- [ ] Authored docs link sang `_generated/` thay vì duplicate bảng
- [ ] `README.md` của app-map phân biệt rõ 2 loại trong index
